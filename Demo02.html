总结Spring Boot相关的注解

1、Java的元注解

  • @Retention 注解存活时间

    • RetentionPolicy.SOURCE 表明注解只在源码阶段保留
    • RetentionPolicy.CLASS 表明注解被保留到编译运行的时候，不会加载到JVM中。
    • RetentionPolicy.RUNTIME 表明注解可以留到程序运行的时候，会被加载到JVM中。

  • @Documented 表示注解内容会被Javadoc工具提取成文档。

  • @Target 表示注解用于什么地方。

    • ElementType.FIELD 表示对字段、枚举常量的注解
    • ElementType.METHOD 表示对方法的注解
    • ElementType.PARAMETER 表示对参数的注解
    • ElementType.CONSTRUCTOR 表示对构造函数的注解
    • ElementType.LOCAL_VARIABLE 表示对局部变量的注解
    • ElementType.ANNOTATION 表示对注解类型的注解
    • ElementType.PACKAGE 表示对包的注解
    • ElementType.TYPE 表示对接口、类、枚举、注解等任意类型的注解

  • @Repeatable 总标签

2、Java的基本注解

  • @Deprecated 用来标记过期的元素

  • @Suppress Warnings 表示阻止警告的

    • deprecation 忽略过时的警告
    • unchecked 忽略执行了未检查转换时的警告
    • fallthrough 忽略switch程序块直接通往下一种情况而没有break的警告
    • path 忽略路径、源文件路径不存在的警告
    • serial 忽略可序列化的类缺少serialVersionUID定义时的警告
    • finally 忽略任何finally子句不能正常完成时的警告
    • all 表示忽略所有情况的警告

  • @Override 表示子类要重写父类方法

  • @Safe Varags 表示参数的安全类型

  • @FuctionInterface 指定某接口必须是函数式接口，否则编译出错

3、Spring基础注解

  • @Component放在类的前面，，该类被标注成Spring的一个普通Bean。
  • @Controller标注一个控制器组件类，被用来实现自动检测类路径下的组件并将组件自动注册成Bean。
  • @Service标注一个业务逻辑组件类
  • @Repository标注一个DAO组件类

4、Spring常见注解

  • @Autowired被用来自动装配，可以标注成员变量、方法、构造函数等对象。

  • @Qualifier配合@Autowired用来解决异常BeanCreationException。

  • @Resource可用于标注一个对象的SET方法。注解@Resource相当于@Autowired，@Resource时按名字自动注入，而@Autowired是按照类型自动注入。

    • @Resource的两个属性比较重要，name和type，将name注解为Bean的名字，而将type属性解析为Bean的类型
    • 如果不指定name和type属性，则按照反射机制使用名字自动注入策略

  • @Scope定义Bean的作用范围，也可以通过XML文件中设置Bean的scope属性值

    • singleton 全局有且只有一个实例
    • prototype 每次获取Bean时都有新的实例
    • request 每次HTTP请求都会产生一个新的Bean，仅在当前的HTTP内有效
    • session 每次HTTP请求都会产生一个新的Bean，仅在当前的HTTP内有效
    • global session Bean只局限于Portel全局会话的生命周期范围内。

  • @RequestMapping 指定一个映射路径，可以通过指定的路径访问该类和方法。

  • @RequsetBody是把请求中报文的正文部分自动转换成绑定给方法参数的变量字符串。

    • 只有返回内容不是HTML标记页面而是其他格式的数据才能使用@RequestBody注解

  • @ResponseBody将内容或者Java对象转换成响应报文的正文返回。

  • @Param表示对参数的解释，一般写在注释里面。

  • @JoinTable用于标识Java类和数据库表的映射关系，可以标识列、主键等。

  • @Transational是Spring对事务的注解，自动被注册称为事务。

  • @Syschronized表示实现Java的同步机制，相当于加同步锁。

  • @ModelAttribute声明在属性上，表示属性值在model里queryBean，返回字被保存在model里面。

  • @Cachable表明一个方法的返回值应该被缓存。

  • @CacheFlush声明一个方法清空缓存的触发器

  • @InitBinder在Controller中注册属性编辑器，指定某个属性编辑器只作用于特定的控制器Controller。

  • @Required负责检查Bean中SET方法是否被执行，如果没有被执行，则Spring解析器抛出异常提醒对应的属性，@Required只能标注在SET方法中。

  • @Conditional根据满足某些特定条件创建一个特定的Bean。

5、SpringBoot的基础注解

  • @SpringBootApplication和注解@Configuration、@EnableAutoConfiguration、@ComponentScan注解等价。

    • @EnableAutoConfiguration实现自动配置
    • @ComponentScan扫描组件，可自动发现和装配Bean，并把Bean加入到程序上下文。

  • @RestController是注解@Controller和注解@ResponseBody的集合，表示被标注的对象是Rest风格的Bean并且返回值直接填入HTTP响应正文返回给用户。

  • @JsonBackReference可以用来解决无限递归调用问题。

    • 注解@RepositoryRestResouce配合spring-boot-start-data-rest使用，用于创建RESTful入口点。
    • @Import用来导入其他配置类
    • @ImportResource用来加载XML配置文件。

  • @Bean标注方法等价于XML配置中的Bean。

    • @Value注入Spring Boot配置文件中application properties中配置的属性值。
    • @Inject等价于默认的@AutoWired，只是没有required属性。

  • 条件注解

    • @ConditionalOnBean（配置某个特定的Bean）
    • @ConditionalOnMissingBean（没有配置特定的Bean）
    • @ConditionalOnClass（class path目录里有特定的类）
    • @ConditionalOnMissing（classpath中缺少指定的类）
    • @ConditionalOnExpression（给指定的SpEL表达式计算结果为true）
    • @ConditionalOnJava（Java的版本匹配特定值或者一个范围值）
    • @ConditionalOnJndi（参数中给定的JNDI位置必须存在一个）
    • @ConditionalOnResource（class path中有指定的资源）
    • @ConditionalOnNotWebApplication（不是一个Web应用程序）

6、JPA注解

  • @Entity表明被标注的对象是一个实体类，注解@Table(name=" ")指出实体对应的表名，这两个注解一般一起使用，但是如果表名和实体类名相同，则@Table可以省略。

  • @MappingSuperclass注解标注父类，但是@MappingSuperclass标注后不能再有@Entity和@Table注解。

  • @NoRepositoryBean一般用作父类的Repository，有这个疏解Spring不会实例化该Repository。

  • @Column标识实体类中属性和表中字段的对应关系，如果注解@Column的字段名和列名相同，则可以省略。@Column有10个可选属性

    • name 数据库中字段名称
    • unique 字段是否唯一标是默认为false
    • nullable 该字段是否可以为空
    • table 定义包含当前字段的表名
    • length 只有字段类型为Varchar时属性才有效，默认255个字符
    • precision 表示数值的总长度
    • scale 表示小数点所占的位数

  • @Id用于声明一个实体类属性映射为数据库的主键列，该属性通常置于书香声明语句之前，可与声明语句同行，也可卸载单独的行上，@Id标注也可以至于属性getter方法之前。

  • @GeneratedValue用于标注主键的生成策略，通过属性strategy指定策略。可供选择的策略有IDENTITY （Id自增长）、AUTO、SEQUENCE（序列产生主键）等。

  • @Transient表示被标注的属性不是一个到数据库映射的字段映射，对象关系映射框架将忽略该属性。

  • @JsonIgnore的作用是JSON序列化时将JavaBean中的一些属性忽略掉，序列化和反序列化都受影响。

  • JoinColumn（name=“lohinId”）表示一张表只想两一个表的外键。

    • OneToOne
    • OneToMany 会产生中间表
    • ManyToMany
    • ManyToOne 不产生中间表

7、异常处理注解

  • @ControllerAdvice包含注解@Component，可以被扫描到，统一处理异常。
  • @ExceptionHandler（Exception.class）用在方法上面表示遇到这个异常就执行所注解的方法。

8、注解配置解析和使用环境

  • @PreUpdate用于为生命周期事件指定回调方法，用于实体类、映射超类或回调监听器类的方法。@PreUpdate不允许更改实体。
  • @PrePersisit帮助在持久化之前自动填充实体属性。@PrePersist事件在调用persist（）方法后立刻发生 ，此时数据还没有正真插入数据库。可以用来在使用JPA时记录一些与业务无关的字段，如最后更新时间。
  • @PostLoad在Entity被映射之后被调用，注解@EntityManager.Find()或getreference()方法载入一个实体后，执行JPA SQL查询后或者EntityManager.refresh()方法被调用后执行。
  • @PreRemove和注解@PostRemove事件的触发有删除实体引起。注解@PreRemove事件在实体从数据库删除之前触发。注解@PostRemove事件在实体从数据库删除后触发。
  • @NoArgsConstructor提供一个无参的构造方法。注解@AllArgsController提供一个全参的构造方法。
